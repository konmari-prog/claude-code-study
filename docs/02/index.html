GAS（Google Apps Script）とは

code一言でいうと

Googleサービスの裏側で動く無料のプログラミング環境。JavaScriptベースで、サーバーもインストールも不要。

check_circleできること

  • スプレッドシートの自動処理
  • フォーム回答の自動集計
  • メールの自動送信
  • WEBアプリの公開
  • 定期実行（トリガー）

lightbulb スプレッドシート、フォーム、Gmailなどと直接つながるのが最大の強み。外部APIの設定なしでGoogleサービスを自在に操作できる。

今日の実装フロー

1

clasp インストール & ログイン

$ npm install -g @google/clasp $ clasp login

GAS APIの有効化も忘れずに: script.google.com/home/usersettings

2

Googleスプレッドシートを新規作成

空のスプレッドシートを1つ作成する。これがアンケート集計の母体になる。

3

スクリプトIDを取得

メニュー「拡張機能」→「Apps Script」を開く。URLの https://script.google.com/home/projects/ここがID/edit をコピー。

4

Claude Codeでスキルを呼び出す

# Claude Code を起動して... "アンケート作って" # スキルが質問してくる # Q1: どんなアンケート？ → 例: セミナーの満足度調査 # Q2: 質問項目を確認 → OK / 修正 # Q3: スクリプトID → 貼り付け
5

ファイルが自動生成される

Code.gs / index.html / help.html / appsscript.json / .clasp.json がローカルに出力される。

6

clasp push でデプロイ

$ clasp push --force Pushed 4 files.
7

フォーム作成 & ダッシュボード公開

スプレッドシートをリロード → メニュー「アンケート集計」→「フォームを作成」
GASエディタ →「デプロイ」→「新しいデプロイ」→ WEBアプリ → URL取得

6

トラブルシューティング

errorclasp pushが失敗

原因: GAS APIが未有効
対処: usersettings でAPIをオン

error認証エラー

原因: claspのログイン切れ
対処: clasp login を再実行

errorデータが表示されない

原因: シート名の不一致
対処: 「フォームの回答」シートが存在するか確認

error更新が反映されない

原因: デプロイの新バージョン未作成
対処: 「デプロイを管理」で新バージョン発行

support_agent エラーが出たらClaude Codeにエラーメッセージをそのまま貼り付けて相談するのが最速の解決法。
