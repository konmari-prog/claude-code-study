code Google Apps Script（GAS）

Googleが提供する、Googleサービスを自動化するためのプログラミング環境。スプレッドシート、フォーム、Gmail、カレンダーなどをコードで操作できる。

GASのコードを自分で書く必要はありません。Claude Code が書いてくれます。あなたがやるのは「こういうの作って」と伝えることだけ。

なぜ clasp clone が必要？

clasp clone は、スプレッドシートのGASプロジェクトをローカルに「コピー」するコマンドです。これにより .clasp.json が自動生成され、以降の clasp push が正しいプロジェクトに紐づきます。この手順を飛ばすと push 先がわからずエラーになります。

lightbulb

clasp clone を実行したフォルダで Claude Code を開いてください。Claude Code はこのフォルダにコードを生成します。

第7回で kikaku スキルをインストールしたのと同じ手順です。gas-guidelines はGASの鉄則・実行制限を守らせるルール、gas-survey-builder はアンケート集計GASの生成手順を覚えさせるスキルです。両方入れると品質が安定します。

STEP 9
デプロイしてダッシュボードを確認する

GASエディタでWEBアプリとして公開し、ダッシュボードにアクセスします。

  1. スプレッドシートのメニュー「拡張機能」→「Apps Script」でGASエディタを開く
  2. 右上の「デプロイ」ボタン →「新しいデプロイ」をクリック
  3. 種類の横にある歯車マーク →「ウェブアプリ」を選択
  4. 以下の設定にする：
    説明アンケート集計ダッシュボード
    次のユーザーとして実行自分
    アクセスできるユーザー全員
  5. 「デプロイ」をクリック → 表示されたURLをクリック

デプロイ設定の意味

person

自分として実行

スプシのデータを読み取るとき、あなたの権限を使う。他の人がダッシュボードを見ても、スプシへの権限を個別に渡す必要がない。

public

全員がアクセス

URLを知っていれば誰でもダッシュボードを見られる。アンケート結果をチームや参加者に共有するために必要。

「新しいデプロイ」と「デプロイを管理」の違い

新しいデプロイ

新しいURLが発行される。今回のように初めてWEBアプリを公開するときに使う。

デプロイを管理

既存デプロイの鉛筆マーク → バージョンを「新しいバージョン」に変更 → デプロイ。URLはそのままで、コードだけ更新できる。

lightbulb

コードを修正した後は「デプロイを管理」からバージョン更新すればOK。URLは変わりません。Webhookのように外部サービスにURL登録している場合も、登録し直す必要はありません。

smart_toy

実は、デプロイもClaude Codeに任せられる

今回は仕組みを理解するために手動でやりましたが、clasp deploy コマンドでデプロイも自動化できます。

つまり Claude Code に「デプロイまでして」と言えば、コード生成 → push → デプロイ まで全部やってくれます。慣れてきたら試してみてください。

lightbulb

ダッシュボードが表示されて、回答データがグラフに反映されていれば完成です。このURLをブックマークしておくと、いつでもダッシュボードを確認できます。
